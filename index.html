Using Gerrit for Change Control

Jan 27th, 2013

As an OpenStack contributor I’ve had to learn to use Gerrit to submit changes to the Horizon project. OpenStack uses Gerrit to make sure any changes to source code go through a review and testing process before being committed to the official Github repository. Once a change has been reviewed by a number of reviewers and approved by a core reviewer, Gerrit automatically merges the code to the repo. Gerrit also takes care of running unit tests, static code analysis and gate jobs during the process.

Now that I’ve seen how Gerrit benefits OpenStack, I’m experimenting with using it for internal projects. Gerrit is fairly simple to install, and can integrate with existing repositories on Github or within your private infrastructure.
